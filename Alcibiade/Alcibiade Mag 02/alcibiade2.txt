-=L'ASM ???=-

-=pourkoi apprendre comment ca marche ??=-

-=forme d'un programme en assembleur=-
voici comment ce presente un listing (code source) en assembleur =>
(le ";" sert a entrer un commentaire in n'est poa pris en compte kand vous
assemblerez le prog) 

title   votre_titre    ;ca c'est le titre (poa indispensable)      
dosseg                 ;modele de segmentation dos
.486
.model small           ;petit model 
.stack 100h             ; 256 octet pour la pile
.data                  ;segement de donnee
  ;on mettera ici les donnees (variables)

  ;on foutra ici le code

donc voila a koi represente en gros comment une source de prog assembleur
je vais maintenant essayer d'expliquer tout ces trucs =>

.486 => active les instruction du 486 on peut aussi trouver .8086,.186,.286,.386
        .8087,.287 et .387


# model =>cette instruction definit le modele de memoire ki va etre utilise
par le prog, on peut choisir comme model:

-tiny: code+segement de donne peuvent occuper jusqu'a 64Ko

-small: encore 64Ko pour le code et le donnees, mais ce model donne 
des fichier .com

-medium: le segement de donees peut prendre jusqu'a 64Ko mais le code peut
occuper jusqu'a 640Ko

-compact: contraire de medium, les donees peuvent occuper 640Ko mais le code
64Ko, ce type de model ne donne que des fichier .exe

-large: code+donnees peuvent prendre jusqu'a 640Ko,on peut aussi trouver
huge comme model,c'est la meme chose que large.

# stack => c'est la pile, ca sert a empiler des donees (comme des assiettes)
on verra plus tard comment s'en servir

# data => c'est la ke l'on met les variables,les donnees dont on va se servir
par exemple:

text  db  "un_texte","$"

voila on a mis un texte dans la variable text, le "$" signifit ke text 
contient un code ascii (des lettres)
au debut du code (.code) on trouve:   mov ax,@data
                                      mov ds,ax
ca sert a initialiser le segement de donee, ca indique ou ce trouvent les variables

# code => c'est la ke l'on met le code,il est compose de d'interruptions,par
exemple vous voulez afficher le contenu de la varible text a l'ecran, il 
existe une fonction prevu :
        mov   ah,09h           ;met 09h dans ah
        mov   dx,offset text 
        int   21h
cette fonction est la fonction 9h de l'interuption 21h.
mov sert a mettre quelque chose dans quelque chose: ici, on met un numero
de fonction dans un registre.



-=les registres=-
vous avez surement remarque dans le bout de code au dessus des trucs bizares:
al et dx, ce sont des registres, voici les principaux registres et 
leurs utilisation:

° les registres de donnees du processeur sont:al,bx,cx,dx
 ces registres sont composes de 16 bits depuis le 80386 on a des registres 
 en 32 bits les voici:eax,ebx,ecx,edx (il suffit de rajouter e devant les 
 anciens registres)

° utilisation des registres:
-ax sert a transmettre les numero de fonction kand on apelle une interruption
-bx sert a une forme particuliere d'addressage 
-cx sert pour les boucles, exemple:

        mov   cx,30          ;cx est egale a 30
boucle: mov   ah,09h         ;apelle la fonction 09h
        mov   dx,offset text 
        int   21h
        loop  boucle         ;incremente cx (cx=cx-1) puis va a boucle 

si vous avez bien compris, le bout de code au dessus sert a ecrire 30 fois
ce k'il y a dans la variable text, l'equivalent en pascal est:
    for i:=1 to 30 do begin
      writeln(text)
    end;

-dx sert dans certaines divisions et multiplications,dx peut aussi tout 
simplement sevir a stocker des donnees.
 



-=la pile=-
comme vous l'avez vus il n'y a poa enormement de registres, ils servent 
d'ailleurs tous a poa mal de choses, il faut donc trouver un moyen de ranger
des information ailleurs ke dans ces registres.la solution:la pile...
la pile est un moyen de stocker des informations en les empillant.
par exemple vous voulez garder le contenu de ax mais vous voulez utiliser
un fonction ki demande ax, vous allez donc empiler ax, appeler votre 
fonction puis depiler ax, voici l'exemple:

   push ax      ;met ax dans la pile
   mov  ax,01h  ;apelle la fonction 01h de l'int 33h
   int  33h 
   pop  ax      ;depile ax  

push set a mettre le contenu d'un registre dans la pile,pop sert
a mettre la valeur du dessus de la pile dans un registre.
attention => la pile marche d'une maniere speciale,par exemple: 

mov  ax,5  ;ax=5
mov  dx,10 ;dx=10
push ax    ;met le contenu de ax dans la pile
push dx    ;met le contenu de dx dans la pile
            
voici l'organisation de la pile apres ce code:
 --pile--
 _________
 |  5    |
 |  10   |
 |       |
 _________

si on fait pop dx, dx contiendra la valeur du dessus de la pile (5), poa 
forcement celle a laquelle il etait egale avant mais la derniere valeur 
enpiller.
on peut se servir de cette particularite pour echanger les valeurs de 2 
registres, par exemple:
mov  ax,5    ;ax=5
mov  dx,10   ;dx=10
push ax      ;empile ax
push dx      ;empile dx
pop  ax      ;depile en ax (on a donc maintenant dans ax la derniere valeur 
             ;empile donc celle de dx 
pop  dx      ;depile en dx (l'ancienne valeur de ax est maintenant dans dx)

ATTENTION: le stack overflow
c'est quand la pile depasse la memoire qui lui etait destine, consequance:
(sous win$uck) une belle boite avec marque fatal-error
vous vous demandez peut etre pourkoi je vous pale de ca,c'est tres simple: 
par exemple kand on craque une protection de cd il arrive que l'on suprime
tout un sous programme.Peut etre que ce sous programme contenait des pop
ou des push,si c'est le cas,ne les suprimez poa sinon,vous avez 99% de 
chances que le cracke foire.

voila pour la pile...




-=les principales instructions=-
on a deja vu mov (qui met une valeur dans un registre), push (qui empile le
contenu d'un registre) et pop (qui depile la valeur sur le dessus de la pile)
passons maintenant aux autres instructions:

(j'ai classe les instructions par ordre alphabethique)

               Comment recharger sa carte téléphonique ?

Ba en fait c pas difficile mais le mec qui a trouvé ça a vraiment bcp d'imagi
nation !

En + il ne faut qu'un minimum de matos: 

- une carte téléphonique avec quelques unités.
- du scotch.
- un congelateur.

Prenez une carte téléphonique pratiquement terminé (du genre 5 unités).
Mettez du scotch sur la puce et repassez avec un truc assez pointu les lignes
de la puce.Faites gaffe si vous prenez un couteau de ne pas crever le scotch.
Ensuite coloriez au crayon de papier (assez gras) l'endroit où le scotch re
couvre la puce.Puis enfin foutez la carte au congélo pendant quelques jours...
Au fur et à mesure la carte va se recharger ! 
Attention aux cartes trop vieilles car ça peu foirer.Attention aussi de ne pas
prendre une carte qui n'a plus n'unités car dans ce cas là , la puce est grillée.
