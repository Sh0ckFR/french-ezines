 6  -/-   Initiation : Se connecter a Internet sous Linux avec eZppp   -/-
----------------------------------------------------------------------------- 


Votre serviteur, le chat du dr gang, va vous expliquer comment se connecter
a Internet sous Linux avec eZppp (Client xWindows) : c'est point hardu.
En fait c'est meme beaucoup plus simple que sous un certain OS, repandu,
mais que je ne m'abaisserais point a citer.

 - 1 st : S'assurer que votre serviteur le root a bien compile les
protocols PPP dans le Kernel.
Le support rezo et le TCP-IP etant integres en standard dans Linux, vous
ne devriez pas avoir de problemes.

 - 2 nd : Installer EZPPP dans son XWindows favori (moi je preferes
Enlightenment).
Decompresser l'archive : gunzip ezppp-1.0BX-i386.tar.gz
pour l'archive que vous aurez dans mon lien (v. dessous)
ou : tar xvf ezppp-1.0BX-i386.tar pour les .tar.

 - 3 rd : Config EZPPP
                Rentrer le nom votre connexion piratee
                Le NO de phone de votre provider pirate
                Ensuite allez parametrer votre script :
                        Choississez 'Expect' puis 'ogin:',
                        puis 'Send' puis 'mon_login_chez_wanadoo'
                        de la meme facon, entrez votre
                        pass (expect 'assword', send..)
                Configurez les DNS dans l'onglet DNS.


 - 4 th : Si vous n'etes point logge en root, va falloir setuper les
 permissions

Permissions de ROOT a votre daemon PPP : chmod +s pppd
Permission acces Modem pour un modem en port com 3 : chmod 666 /dev/cua2
        Si votre modem est en com 1, c'est /cua0
        (Port COM number = x  =>  cua number = x-1)
Permission d'acces au vos setups DNS : chmod 666 /etc/resolv.conf


 - 5 th : Si ca foire :
        J'ai lu dans un How-To que si ca foirait, fallait virer le contenu
du fichier /etc/ppp/options (fichier par defaut), comme ca, ca interferes pas
avec celles d'EZPPP, laisser le fichier vide.

        Pareil : sous Slackware 96, le fichier /usr/sbin/pppd
est un script qui executes /usr/sbin/pppd-2.1 ou /usr/sbin/pppd-2.2
selon votre version du kernel. Donc vous devez aller a Config->General
et changer le path vers l'un ou l'autre. Si vous ne le faites pas,
EZPPP ne raccrochera pas correctement.

        Autre chose : Netscape a son propre solveur de DNS qui accede a
/etc/resolv.conf, donc si vous utilisez Netscape et que ca foire,
faudra lancer Netscape apres avoir lance EZPPP

        Perso j'ai eut aucun problem...


 - 6 th : Vous avez plus qu'a lancer votre connection, et la, surprise :
        votre provider de merde est etonnamment plus rapide ??! C'est
        juste parce que Linux gere en natif le TCP-IP et manage les
        acces rezo de facon potable...

        Si vous utilisez Lynx pour le Web, pas besoin de config, suffit
        de le lancer. Pareil pour Netscape.

   - 7 th : le FTP
        Vous allez avoir une surprise aussi : MC (Midnight Commander),
        que je vous conseille vivement si vous l'avez pas, gere les     
        FTP comme si c'etait un repertoire normal, vous avez qu'a le    
        mounter, em choisissant cette option (dans le menu fichier,
        je crois) et la vous tapez F1 pour savoir comment enter
        le nom du FTP avec le pass et les param que vous voulez,
        c'est un truc du genre : ftp.warez.org:pass:login.
