Comment decoder le code de partage des fichier ?

quand on hack un ordi sous windows, on cherche d'abord si il as pas partager ses fichier, il arrive souvent que des fichiers sois partager mais helas, proteger par un pass ... :( c pas l'extase ... et bin on va voir comment decoder cette saloperie de code de merde !

d'abord y faut savoir que le pass est stocké dans la base de registre et fait maximum 8 lettre
c'est stocké dans (attention acroché vous :p )
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Network\LanMan\le_nom_du_repertoire
exemple pour ceux qui ont pas compris, si c'est le repertoire "prout" qui est partagé :
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Network\LanMan\prout
si c'est un disk, comme par exemple le C :
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Network\C

mettons comme code "la vache" dans la bdr on va avoir :
79 DB 6D F0 12 EA 9C 2F
on va mettre tous ça dans un tableau :

code : L A   V A C H E
en hexadecimal : 6C 61 00 76 61 63 68 65
crypté : 79 DB 6D F0 12 EA 9C 2F

operation XOR
clée de cryptage : 15 BA 6D 86 73 89 F4 4A

en temp normal le code ascii de la barre d'espacement est 20, mais là c'est 00
donc la clée de cryptage est 15 BA 6D 86 73 89 F4 4A et vous pouvez essayer avec d'autre code, vous obtiendrez toujours la meme clée, je sais pas si c'est la meme clée pour les autres versions de windows, moi j'ai testé sous win 98 premiere edition
